## 📂 Organização do Repositório

Para garantir a escalabilidade e a limpeza técnica, este repositório segue a estrutura:

1. **Modalidade:** Identificação se a atividade foi realizada em Dupla ou Trio;
2. **Nomenclatura Semântica:** Pastas separadas por atividade e tema.
   * *Exemplo:* `PM_ATV1_LAYOUT_BASICO`
3. **Documentação de Entrega:** Cada pasta contém o código-fonte e um arquivo `README.md` específico com:
   * Objetivo do aplicativo;
   * Data de entrega;
   * Nomes dos integrantes e professores;
   * Instruções de execução (Emulador ou Dispositivo Físico).
